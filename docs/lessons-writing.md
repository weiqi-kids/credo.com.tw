## 一、文體：模型會犯什麼（全部實際發生過）

| 失敗模式 | 實例 | 教訓 |
|---|---|---|
| （）旁白根深蒂固 | 明文禁止後仍寫出「（化名，原告）」「（臺灣橋頭地方法院114年度…）」 | 禁令要放 prompt 最前段＋聲明「每條都有程式檢查」；仍需正則兜底 |
| 「不是…而是」會變形 | 被禁後改寫成「不在…而在…」逃過正則 | 語意型變體只有第二個模型（獨立查核）抓得到，正則永遠追不完 |
| 腦補細節 | 判決只寫「房屋」→ 寫成「透天厝」；隱去的鄉鎮 → 猜「虎尾鎮」；沒寫排行 → 編出「大女兒、二兒子」；整整一年 → 「過了一年多」 | 忠實鐵則要明示「寧可短，禁止補」＋列出常見腦補類型（親屬關係/時間/地點/動機） |
| 推論包裝成認定 | 「法院發現已改名」（實為起訴時已載）、「依民法第474條判決」（判決未引條號） | 要求推論一律帶「可能/推測」；查核 prompt 專項檢查「將推論寫成法院明文」 |
| 化名不徹底 | facts 用化名、court 段殘留真名「陳冠瑋」 | 化名規則要逐欄位講：「全部欄位含 court 一律化名」 |
| 列點式起手 | 「點出兩個實務重點。第一…第二…」＝首先其次的變體 | 查核官已能識別「高度相似的變體」，保留其自由裁量空間 |
| 引用偷懶 | 漏列判決真正的關鍵法條（仲裁法 37 條、民法總則施行法 4 條）、來源抓部落格（vocus/代書網站） | 法條完整性納入查核；資料來源 official-only（gov.tw 系）白名單 |

## 二、文體：有效的正面手段

- **口語標題公式**成立：「爸媽過世後遺產怎麼分？」勝過法條式標題；法律名詞放內文與 FAQ
- **敘述句引法條**：「依民事訴訟法第 51 條，債權人可以聲請…」，禁「XX（民訴法51條）」
- **律師語感基準**：對客戶說話的口吻、句子短、動詞開頭、每段一個重點、對比句用「是Y，而非X」
- 25 篇鎮站文結構（檢查表＋破迷思＋費用行情＋FAQ）與錦囊結構（行動清單＋時限表＋不夠用段）本身可留用——**未達等級的是文字肌理，不是骨架**

## 三、產線：把關機制的實測數據

- **過件率約 6–25%**（17 次嘗試 1 篇過、DRY 輪 9 次 2 篇過）——查核官寧可錯殺是刻意設計，量產瓶頸在改寫品質而非把關

- 全站 audit-ai-tone 是 CI 硬 gate：違規文章推不上 main

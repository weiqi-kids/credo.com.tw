# 鎮站文產文規格（Phase 2，25 篇統一標準）

每篇是該主題的「收單主力」：接住「正要辦這件事」的搜尋者，格式固定如下。

## frontmatter（一字不差照這個結構）

```yaml
---
title: （口語標題，含主要關鍵字，禁法條式標題；例「簽約前必看：7 個最常讓中小企業吃虧的條款」）
description: （120–150 字摘要，含 2–3 個關鍵字，會成為 meta description）
date: "2026-07-04"
category: corporate-legal | personal-asset   # 依主題所屬分類
topic: （主題 slug，見 docs/content-taxonomy.md）
kind: pillar
draft: true
keywords: （逗號分隔 4–6 個台灣搜尋詞）
faq:
  - q: （口語問題，像真人會問的）
    a: （80–150 字直接回答，可獨立被引用）
  # 共 3–5 題
---
```

## 內文結構（1500–2500 字，Markdown）

1. **開場**（100–150 字）：說中讀者處境（「正要簽約/剛收到存證信函的你」），一句話講清楚看完能得到什麼
2. **預防檢查表**（主體）：H2「XX 前必看的 N 個檢查點」，每點 H3＋說明，N=5–8
3. **常見迷思破解**：H2，2–3 個「很多人以為…其實…」
4. **費用與流程行情**：H2，講「大概多少錢、多久、找誰辦」；數字一律寫「行情約/通常」，不得寫死保證數字
5. **什麼時候該找律師**：H2，具體情境 3–4 個（自然導向諮詢，禁招攬用語）
6. **資料來源**：H2，列出引用的法規「法規名 第X條」與官方資料（先不放連結，連結由審稿程式統一生成驗證）

## 鐵則（違反即退稿）

- **AI 味語言全面禁止**（完整清單＝`scripts/audit-ai-tone.mjs`，CI 硬性擋）：（）括號旁白、「不僅…更」「不是/不只是…而是」「並非…而是」、值得注意的是、換句話說、總的來說、綜上所述、扮演…角色、至關重要、不可或缺、隨著…的發展、讓我們、不妨、希望本文、燈塔/見證式拔高、首先其次最後、模糊引用（研究顯示/專家認為/有人說）、模板化第一人稱開頭。對比句改用「是Y，而非X」或正面直述。法條引用用敘述句：「勞動基準法第 17 條規定資遣費計算方式」。

- **只用中華民國（台灣）現行法規**；禁外國法、中國法規、比較法
- **所有數字（稅額/期限/費用）必須先 WebSearch 查證 2026 年現行值**，不確定就寫「以主管機關公告為準」；資料來源段列出查證來源網址
- 引用法條寫「法規全名＋條號」（如「勞動基準法第 17 條」），**禁止自己編法條連結網址**
- 禁：勝訴保證、誇大招攬用語（律師倫理）、「本所/本律師」自稱（署名另有機制）
- 內文內鏈用 Markdown 根路徑：企業類文末段連 `[企業安全解決方案](/companysafe/)`、個人類連 `[個人安全解決方案](/naturalpersonsafe/)`；禁 raw HTML 標籤
- 語氣：專業但說人話，繁體中文台灣用語，段落短（≤4 行）
- 不重複其他主題的內容範圍（只寫自己主題，相鄰主題點到為止）

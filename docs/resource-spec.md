# 錦囊（實用工具）產文規格

一份錦囊＝一個「角色 × 關鍵時間點」的行動工具：讀者事發當下照著做。

## frontmatter

```yaml
---
title: （口語，含情境關鍵字，例「收到存證信函別慌：拆解、回應、留證的完整行動清單」）
description: （120–150 字，說清楚這份工具幫誰、在什麼時刻、解決什麼）
date: "2026-07-05"
category: corporate-legal | personal-asset
topic: （對應主題 slug，見 docs/content-taxonomy.md）
audience: （給誰用，例「新進員工」「收到信的一方」「家屬」）
moment: （關鍵時間點，例「到職簽署文件時」「收到存證信函當天」）
keywords: （逗號分隔 4–6 個）
faq:
  - q: …
    a: （80–150 字）
  # 3–4 題
---
```

## 內文結構（1200–2000 字）

1. **開場**（80 字內）：直接說「現在該做什麼」，一句話定調
2. **立即行動清單**：H2「現在就做的 N 件事」，每件 H3＋為什麼＋怎麼做，N=4–7，按時間順序
3. **文件逐條解說**（有文件的情境）：H2，這個時刻會碰到的文件/條款逐項講「看什麼、什麼情況別簽/別回」
4. **千萬別做的事**：H2，2–4 個常見錯誤動作與後果
5. **時限表**（有法定期限的情境必加）：H2，用表格列「幾天內要做什麼、法源」
6. **什麼情況這份清單不夠用**：H2，3 個該直接找律師的具體訊號
7. **資料來源**：H2，法規全名＋條號，官方來源

## 鐵則（同鎮站文＋加嚴）

- **AI 味語言全面禁止**：完整清單＝`scripts/audit-ai-tone.mjs`，交稿前必須自查（禁（）旁白、不是…而是、值得注意的是、換句話說、總的來說、至關重要、讓我們、不妨、首先其次最後、模糊引用等）；對比句用「是Y，而非X」或正面直述
- 只用中華民國（台灣）現行法規；數字與期限用 WebSearch 查證 2026 現行值；法條寫「法規全名＋條號」敘述句，禁自編網址
- 禁勝訴保證/招攬語/「本所本律師」；禁 raw HTML；內鏈用 Markdown 根路徑
- 語氣：像可靠的朋友在旁邊指路，句子短、動詞開頭、可執行
